이력서 예상 질문
Q) 21년 2월에 졸업 하셨던데 그이후로 지금까지 뭐하시면서 지냈을까요?
A) 구직활동을 하면서 저에게 부족한 자격증에 대한 공부나, 체력관리를 위한 1시간 운동
개발능력을 기르기위해 학원을 찾아 프론트엔드 개발, 백엔드 개발에 대해 교육을 받았습니다.

국비 훈련 진행하면서 전체성적 2위라는 성적으로 수료했던데, 어떤식으로 참여했습니까?
- 훈련 시작이 아침 9시 00분이였기에 매일 20분전에 미리 나와 준비한 뒤, 참여를 했고 훈련종료 후 자습실을 통해 남아 복습 했습니다.
* 예상 파생 질문 : 복습 한 기록이 있습니까? - 블로그와 Git을 통해 내용을 정리하면서 복습했습니다.
최근에는 블로그는 쉬고있지만 취업준비를 하는 내용들을 Git에 저장하고 있습니다.

시킨일 하나는 정말 잘한다고 쓰여있는데, 불합리한 일을 하더라도 불만없이 하겠습니까?
- 저는 회사 내규나 법적으로 어긋나는지 아니면 회사에 해가되는지 먼저 판단해보겠습니다. 어긋나는 일이라고 확인된다면
동료들이나 가까운 선임분께 조심히 물어보고 행동하겠습니다. 만약 부당한 지시란 생각이 저만의 생각이라면, 불만없이 행동하겠습니다.

온라인 강의 사이트를 만들면서 어려웠던 일이나 팀원들과의 불화는 없었나요?
- 어려웠던 일은 수강 후기를 만들던 중, 후기 수정요청을 보내도 페이지에서 변화가 없는 모습을 발견했습니다.
디버깅을 통해 문제가 발생한 위치를 찾아보았으나 별다른 원인이 없었습니다. 그래서 페이지를 살펴보던중 수정요청에서 문제가 있을꺼라는
확신과 함께 추가조사를 들어갔고, 데이터 타입이 명시되어있지않아 발생한 문제였다는점을 깨닫고, 타입을 정해주는것으로 해결할 수 있었습니다.

기술면접 예상 질문

JAVA의 특징을 얘기해 보세요
- 객체 지향언어, 네트워크 및 분산처리 지원, 가비지 컬렉션을 통한 자동 메모리 관리, 멀티 스레드 지원
- 장점 : JVM(Java Virutal Machine; 자바 가상 머신)위에서 동작하기 때문에 운영체제에 영향을 받지 않는다.
	
JVM의 역할에 대해 설명해보세요
- Stack(스택) 기반으로 동작
- 자바 코드를 운영체제에 맞게 해석 및 가비지콜렉션을 통한 메모리 관리

OOP(객체 지향 프로그래밍 | Object-Oriented Programming) 이 무엇인지 설명해 주세요
- 데이터를 객체로 취급하여 프로그램에 반영한 것이며, 순차적으로 프로그램이 동작하는 기존의 것들과는 달리 객체와 객체의 상호작용을 통해 프로그램이 동작하는것을 의미합니다.
* 특징
- 객체지향 프로그래밍은 코드의 재사용성이 높다.
- 코드의 변경이 용이
- 직관적인 코드분석
- 개발속도 향상
- 상속을 통한 장점 극대화

객체지향언어가 무엇인가요?
- 프로그램을 다수의 객체로 만들고, 이들끼리 서로 상호작용하도록 만드는 프로그래밍 언어입니다.
- 캡슐화, 상속, 다형성, 추상화 등의 특징이 있습니다.
* 특징
- 캡슐화 : 데이터와 코드의 형태를 외부로부터 알 수 없게 하고, 데이터의 구조와 역할, 기능을 하나의 캡슐형태로 만드는 방법
- 상속 : 상위 클래스의 모든 것을 하위 클래스가 모두 이어 받는 것
- 다형성 : 상속과 연관된 개념으로 한 객체가 다른 여러형태(객체)로 재구성 되는 것
- 추상화 : 객체의 공통적인 속성과 기능을 추출하여 정의하는 것

자바의 메모리 영역
- 메서드(static변수 등에서 사용되는 클래스 정보가 들어가는 영역)
- 스택(지역변수, 함수(메서드) 등이 할당되는 LIFO(Last In First Out)영역)
- 힙(new로 생성된 객체가 들어가는 영역)

추상 메소드, 추상 클래스
- 추상 메소드 : 메서드로 정의만 되어있고 구현은 되어있지 않는 메서드
- 추상 클래스 : 추상메서드를 적어도 하나 이상 가지고있는 클래스

프로세스(Process)와 쓰레드(Thread)의 차이는?
- 프로세스 : OS가 메모리 등 자원을 할당한 프로그램[서로간 독자적인 메모리 영역을 가지고있기에 서로간 메모리 공간을 공유하지 못한다.]
- 쓰레드 : 프로세스 내에서 프로세스의 자원을 가지고 실제로 작동하는 일꾼[독자적인 Stack메모리 가짐]

컬렉션프레임워크(Collection FrameWork)란?
- List 인터페이스 : 배열과 유사하다, 데이터 추가시 자동으로 길이가 늘어나는 구조, 데이터 중복 허용, 순서가 존재한다.
* 종류 : ArrayList, LinkedList, Vector
- Set 인터페이스 : 중복된 데이터 허용하지 않는다, 순서가 없다
* 객체 내부의 중복된 데이터를 배재하기 위해선 equals 메서드와 hashCode 메서드의 재정의가 필요하다
* 종류 : HashSet, TreeSet
- Map 인터페이스 : Key와 Value 쌍으로 데이터를 저장하며, Key는 중복될 수 없지만, Value는 중복 저장이 가능하다.
* 종류 : HashMap, TreeMap, Properties, Hashtable

Session, Cookie의 공통점과 차이점
공통점 : 사용자의 데이터를 사용한다.
차이점
- Cookie : 클라이언트의 컴퓨터에 저장했다 서버 요청시 네트워크를 타고 서버에 전달되기 때문에 보안에 취약하다.
- Session : 서버에 저장되고 브라우저 단위로 저장된다. 쿠키에 비해 보안성이 좋다.

GC(Garbage Collecton)이란 무엇인가요?
- 자바 애플리케이션에서 사용하지 않는 메모리를 자동으로 수거하는 기능입니다.
- C와 C++에서는 메모리를 직접 할당하고 해제해야하지만, 자바에서는 GC를 이용하여 개발자들이 메모리 관리를 신경쓰지 않아도 됩니다.

Overriding 과 Overloading 이 무엇인지 설명해주세요
- Overriding : 같은 이름의 메소드를 여러개 정의하는것, 매개변수의 타입이 다르거나 갯수가 달라야한다.
- Overloading : 부모클래스로부터 메소를 상속받아 자식클래스에서 재정의하는 것

JAVA의 접근제어자에 대해 설명해주세요
- 접근제어자의 종류로는 public, protected, default, private등이 있습니다.
- public : 모든 패키지, 클래스 접근 허용 | protected : 같은 패키지, 모든 클래스 접근 허용 [다른 패키지의 경우 자식 클래스의 접근 허용]
- default : 같은 패키지 내 클래스만 접근 허용 | private : 같은 클래스 내 접근만 허용
* 사용 이유 - 클래스 내부에 선언된 데이터의 부적절한 사용으로부터 보호하기 위해서 접근 제어자를 사용

디자인패턴이 뭐있고 어떤 내용인지 설명
- 소프트웨어를 설계할 때 특정 맥락에서 자주 발생하는 고질적인 문제들이 또 발생했을 때 재사용할 할 수있는 훌륭한 해결책입니다.
- 종류로는 브릿지, 빌더, MVC, 팩토리, 싱글톤 패턴등이 있습니다.
* 브릿지 : 큰 클래스 또는 밀접하게 관련된 클래스들의 집합을 두 개의 개별 계층구조​로 나눈 후 각각 독립적으로 개발할 수 있도록 하는 구조 디자인 패턴
* 빌더 : 복잡한 객체들을 단계별로 생성할 수 있도록 하는 생성 디자인 패턴
* MVC(Model View Controller) : 사용자 인터페이스, 데이터 및 논리 제어를 구현하는데 널리 사용되는 소프트웨어 디자인 패턴
* 팩토리 : 부모 클래스에서 객체들을 생성할 수 있는 인터페이스를 제공하지만, 자식 클래스들이 생성될 객체들의 유형을 변경할 수 있도록 하는 생성 패턴
* 싱글톤 : 객체의 인스턴스가 오직 1개만 생성되는 패턴

TypeScript가 무엇인지 설명할 수 있나요?
- TypeScript는 JavaScript의 슈퍼셋으로 자바스크립트의 기본 문법을 가졌으며 추가로 TypeScript만의 문법이 추가된 언어입니다.
- 특징으로는 데이터의 타입을 미리 설정하기 때문에 코드작성 단계에서 오류를 바로 확인할 수 있어 실행속도가 빠르다는 장점이 있습니다.

Restful api가 뭔지 설명해주세요
- RESTful API는 두 컴퓨터 시스템이 인터넷을 통해 정보를 안전하게 교환하기 위해 사용하는 인터페이스입니다.
- 장점으로는 확장성이 뛰어나 REST가 클라-서버 상호작용을 최적화하기 때문에 효율적으로 크기 조정이 가능합니다.
* 동작과정 ? 클라이언트 -> 서버요청 | 서버[클라이언트 인증 및 요청권한 확인] | 서버[요청 수신후 내부 처리] | 서버 -> 클라이언트 응답반환

Request 전송 방식 종류와 간략하게 설명
- GET, POST, PUT, DELETE 등의 Request 방식이 존재
* GET : 클라이언트에서 서버에 데이터를 요청할 때 사용, 전달되는 데이터가 보인다
* POST : 클라이언트에서 서버에 데이터를 보낼 때 사용, 전달되는 데이터가 안보인다
* PUT : Restful에서 데이터 수정작업을 할 때 사용
* DELETE : Restful에서 데이터를 삭제할 때 사용

DDL 과 DML 의 차이에 대해 설명해주세요
- DML은 데이터 조작어의 약자로 테이블내 데이터를 조회, 삽입, 수정, 삭제가 가능한 명령어 입니다.
* 테이블 내 데이터만 대상으로 CRUD가 가능하다.
- DDL은 데이터 정의어의 약자로 테이블과 같은 데이터 구조를 정의하는데 사용되는 명령어 입니다.
* 테이블 프레임워크만 사용한다[Create, Alter, Drop]

Spring FrameWork란 무엇인가요?
- 자바(JAVA) 플랫폼을 위한 오픈소스(Open Source) 애플리케이션 프레임워크(Framework)
* 특징
- 크기와 부하의 측면에서 경량.
- 제어 역행(IoC)이라는 기술을 통해 애플리케이션의 느슨한 결합을 도모
- 관점지향 프로그래밍(AOP)을 위한 풍부한 지원
- 애플리케이션 객체의 생명 주기와 설정을 포함하고 관리한다는 점에서 일종의 컨테이너(Container)라고 할 수 있음
- 간단한 컴포넌트로 복잡한 애플리케이션을 구성하고 설정할 수 있음

Ant나 Maven등 빌드도구가 있는데 Gradle을 사용한 이유가 무엇인가요?
- Gradle은 Groovy 기반으로 빌드 자동화, 개발 지원에 중점을 둔 빌드 도구로서 Ant와 Maven의 장점을 모아 제작된 도구입니다.
- JVM에서 동작하는 Groovy를 기반으로 동작하기 때문에 자바 개발자로서 쉽게 익힐 수 있어서 사용했습니다.

Spring Data JPA는 무엇인가요?
- Repository를 제작할 때마다 반복적으로 작성되는 CRUD 메서드가 존재합니다.
이때, 반복 작성되는 메서드를 자동화하여 기본적인 CRUD 메서드를 제공하는 라이브러리입니다.

Servlet 과 JSP의 차이
Servlet : Container가 이해할 수 있게 순수 자바코드로만 구성된것(HTML in JAVA)
JSP : HTML코드에 JAVA코드를 블록화하여 삽입한 것(JAVA in HTML)

JSP란 무엇인가?
- Java Server Page의 약어로, HTML 코드에 JAVA 코드를 넣어 동적웹페이지를 생성하는 웹어플리케이션 도구
- JSP 가 실행되면 자바 서블릿(Servlet) 으로 변환되며 웹 어플리케이션 서버에서 동작되면서 필요한 기능을 수행하고 생성된 데이터를 웹페이지와 함께
클라이언트로 응답합니다.

Git, CVS, SVN 관련 질문
CVS와 SVN에 대해 아는대로 설명하세요

CVS : 1990년 출시, 중앙에 위치한 Repository에 파일을 저장하고, 모든 사용자가 파일에 접근 할 수 있도록 설계
SVN : CVS의 단점을 보완하여 2000년 출시, 언제든지 원하는 시점으로 복구가 가능하다.
- 공통점 : 형상 관리 툴
* 형상관리 : 소프트웨어 개발 프로세스 각 단계에서 소프트웨어의 변경점을 체계적으로 추적하고 관리하는 활동

Git이 무엇인지 설명하세요
- 개발자가 중앙 서버에 접속하지 않은 상태에서도 코딩 작업을 할 수 있도록 지원하는 버전 관리 시스템
* 특징
- 여러 가지 작업을 동시에 진행하는 경우, branch를 적용하여 다른 기능을 개발한 뒤 merge를 통해 다시 합칠 수 있다.
- 일시적인 서버 장애가 있어도 개발을 계속할 수 있다.
- 서버와 클라이언트 뿐인 기존 형상관리에 비해 분산처리 구조를 유연하게 세울 수 있다

내가 가진 기술이 아닌 회사에서 원하는 기술을 요구할 때? - 저희는 ~~를 사용하는데 ~~를 사용하시는법은 아시나요?
- 저는 ~~를 사용해 본 경험은 없지만 프론트엔드 개발과 백엔드 개발에 관해 배웠던 경험을 바탕으로
